# 01. 프로젝트 개요

## 🎯 프로젝트 비전

**로또탐정**은 로또 번호 분석 및 생성을 위한 풀스택 웹 서비스입니다.

### 해결하고자 하는 문제
- 로또 번호 선택 시 데이터 기반 의사결정 부재
- 분산된 로또 통계 정보
- 재미없는 번호 선택 경험

### 제공하는 가치
- 📊 **데이터 기반 분석**: 역대 당첨 번호 통계 및 패턴 분석
- 🎰 **다양한 생성기**: 랜덤, 패턴 기반, 3D 시뮬레이션
- 🎮 **게이미피케이션**: 포인트, 출석, 레벨 시스템

---

## 👥 타겟 사용자

| 페르소나 | 특징 | 주요 사용 기능 |
|---------|-----|--------------|
| **캐주얼 플레이어** | 주 1회 로또 구매 | 랜덤 생성기, 저장 기능 |
| **분석형 유저** | 통계/패턴 관심 | 분석 페이지, 필터 조합 |
| **엔터테인먼트 추구** | 재미 우선 | 3D 시뮬레이션, 게이미피케이션 |

---

## 📱 핵심 기능 요약

### 로또 분석
- 회차별 당첨 번호 조회
- 번호별 출현 빈도 통계
- 연속 번호, 홀짝 비율, 합계 분석
- Hot/Cold 번호 추천

### 번호 생성
- **랜덤 생성기**: 순수 랜덤 또는 조건부 랜덤
- **패턴 생성기**: 고정수/제외수, AC값, 합계 범위 필터
- **3D 시뮬레이션**: 실제 추첨기를 모방한 물리 시뮬레이션

### 사용자 시스템
- 이메일 인증 + 관리자 승인 가입
- 포인트 & 레벨 시스템
- 출석 체크 & 연속 출석 보너스
- 번호 저장 & 내역 관리

## 🔧 바이브코딩으로 만든 이유

### 왜 바이브코딩?
1. **빠른 프로토타이핑**: 아이디어 → 동작하는 코드가 빠름
2. **풀스택 커버리지**: 프론트/백엔드/DB 모두 가능
3. **학습 곡선 단축**: 새로운 기술 스택도 빠르게 적용

### 실제 경험
- **개발 기간**: 약 2주 (풀타임 아님)
- **주요 도구**: Claude, Cursor, Supabase MCP
- **겪은 어려움**: UI 과다 꾸밈, RLS 권한, 도메인 검증

## 🔧 바이브코딩 팁

### 이 단계에서 중요한 것
- **도메인 지식 정리**: 로또 규칙(1-45, 6개 번호, 보너스 번호 등) 확실히 이해
- **MVP 스코프 정의**: 처음부터 모든 기능 X, 핵심만 먼저
- **기술 스택 결정**: AI가 잘 아는 스택 선택 (Next.js, Supabase 추천)

### 프롬프트 예시
```
로또 번호 분석 서비스를 만들려고 해.
- 1~45 중 6개 번호 선택
- 매주 토요일 추첨
- 역대 당첨 번호 데이터 필요
- 사용자별 번호 저장 기능

Next.js 15 + Supabase로 구현하고 싶어.
FSD 아키텍처로 폴더 구조 잡아줘.
```

- ❌ 한 번에 모든 기능 요청
- ❌ 도메인 검증 없이 진행
