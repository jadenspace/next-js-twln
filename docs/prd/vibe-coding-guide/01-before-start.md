# 바이브코딩 시작 전 준비사항

## 1️⃣ 디자인 시스템 먼저 정하기

### AI Purple Problem 방지
바이브코딩 AI들은 보라색, 그라데이션, 큰 폰트를 남용하는 경향이 있습니다.

**해결책**: 프로젝트 시작 전 디자인 시스템 명시

```
이 프로젝트는 shadcn/ui를 사용해.
- 색상: 흑백 베이스, 포인트 컬러는 노란색만
- 폰트: text-sm 기본, 제목만 text-lg
- 여백: 최소한으로
- 그라데이션, 네온 효과: 사용 금지
```

### 추천 도구
- **shadcn/ui**: 깔끔한 기본 스타일
- **Tailwind CSS**: 유틸리티 클래스
- **클로드스킬-프론트엔드디자인**: 디자인 가이드라인 스킬

---

## 2️⃣ 도메인 지식 정리해두기

AI가 도메인을 잘못 이해하면 치명적입니다.

### 로또탐정 예시
```markdown
## 로또 기본 규칙
- 1~45 중 6개 번호 선택
- 보너스 번호 1개 추가
- 매주 토요일 20:45 추첨
- 1등: 6개 일치
- 2등: 5개 + 보너스 일치

## AC값이란?
- 번호 간 차이값의 고유 개수
- AC = (고유 차이값 수) - 5
- 보통 7~10 사이가 많이 나옴
```

### 왜 중요한가?
- AI가 "보너스 번호도 1~45 중 하나"를 모를 수 있음
- "AC값"을 자동완성으로 잘못 이해할 수 있음
- **본인이 검증 가능해야 함**

## 3️⃣ 폴더 구조 합의하기

AI에게 구조를 명확히 알려주면 코드 위치를 잘 잡습니다.

### FSD 구조 설명 템플릿
```
프로젝트는 FSD(Feature Sliced Design) 아키텍처야.

📁 src/
├── app/          # 라우팅 (Next.js App Router)
├── features/     # 기능별 로직 (auth, lotto, points)
├── entities/     # 핵심 엔티티 (user)
└── shared/       # 공통 유틸 (ui, lib, types)

새 기능은 features/[기능명]/에 만들어줘.
API는 app/api/[기능명]/에 만들어줘.
공통 컴포넌트는 shared/ui/에.
```

## 4️⃣ 기술 스택 명시하기

AI가 추측하지 않게 명확히 지정

```
기술 스택:
- Next.js 15 (App Router, 서버 컴포넌트)
- TypeScript (strict 모드)
- Supabase (Auth + PostgreSQL)
- TanStack Query (서버 상태)
- Zustand (클라이언트 상태)
- Tailwind CSS + shadcn/ui

절대 사용하지 말 것:
- Redux
- CSS Modules
- getServerSideProps (App Router는 다름)
```

## 5️⃣ MVP 스코프 정의

처음부터 모든 기능 요청 ❌

### 단계별 접근
```
Phase 1: 코어 (2-3일)
- 로그인/회원가입
- 로또 번호 랜덤 생성
- 번호 저장

Phase 2: 분석 (3-4일)
- 당첨 번호 조회
- 출현 빈도 통계

Phase 3: 고급 (1주)
- 패턴 분석
- 포인트 시스템
- 3D 시뮬레이션
```
